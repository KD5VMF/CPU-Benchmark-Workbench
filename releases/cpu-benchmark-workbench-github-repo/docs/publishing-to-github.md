# Publishing to GitHub

## What to publish
A good public repository usually includes:
- source code
- README
- license
- screenshots
- contribution guidance
- issue templates
- release notes

## Suggested flow
1. Create a new repository on GitHub.
2. Add these documentation files.
3. Add the source tree and solution file.
4. Commit everything.
5. Push to the default branch.
6. Create a tagged release.
7. Attach your ready-to-run zip build to the release page.

## Good first release contents
- Source code
- MIT license
- A README with screenshots and build steps
- A release zip built from a known-good tag
- A short note about tested Visual Studio / .NET versions
